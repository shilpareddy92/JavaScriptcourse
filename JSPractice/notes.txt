1.JavaScript is a interpreted language. 
It is read and executed line-by-line in real-time by a browser's JavaScript engine (like Chrome's V8) without requiring a separate compilation step beforehand.
However, modern JavaScript engines use a hybrid approach called Just-In-Time (JIT) compilation. 
To make web pages faster and more responsive, the engine translates the JavaScript code into machine code right before it runs

8.Difference between Synchronization and ASynchronization Calls With Examples
SYNCHRONOUS:
Tasks execute one after another.
Next line waits for current line to finish.
Blocking behavior.
Simpler execution flow
ASYNCHRONOUS:
Tasks can start now and finish later.
Next line doesn't wait.
Non-blocking behavior.
Uses callbacks, promises, async/await.

9.promises
In Js, promises are a way to handle asynchronous operations 
in a more organized and  readable manner.
They provide a cleaner syntax for dealing with asynchronous code
compared to traditionla callback-based approches
prmoses are widely used for Tasks like
making network request,reading/writing files and other operations
that dont block the main thread
A promise represenst a value that may not be available yet but will be
resolved at some point in the future.
It can be in one of three states:PENDING,FULLFILLED or REJECTED
Once a promiseis fulfilled or rejected,its considered settled and 
its state cannot change.
