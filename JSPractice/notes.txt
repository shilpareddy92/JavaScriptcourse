8.Difference between Synchronization and ASynchronization Calls With Examples
SYNCHRONOUS:
Tasks execute one after another.
Next line waits for current line to finish.
Blocking behavior.
Simpler execution flow
ASYNCHRONOUS:
Tasks can start now and finish later.
Next line doesn't wait.
Non-blocking behavior.
Uses callbacks, promises, async/await.

9.promises
In Js, promises are a way to handle asynchronous operations 
in a more organized and  readable manner.
They provide a cleaner syntax for dealing with asynchronous code
compared to traditionla callback-based approches
prmoses are widely used for Tasks like
making network request,reading/writing files and other operations
that dont block the main thread
A promise represenst a value that may not be available yet but will be
resolved at some point in the future.
It can be in one of three states:PENDING,FULLFILLED or REJECTED
Once a promiseis fulfilled or rejected,its considered settled and 
its state cannot change.

13.What is Promise.race()?
Promise.race() returns the result of the first promise that settles.
"Settles" means either:
✅ Resolves (success)
❌ Rejects (failure)
It doesn't wait for the others.

17.Async await
a.Initially, JavaScript handled asynchronous operations
using callback functions.
b.A callback is a function passed as an argument to another function 
and executed after an asynchronous task completes.
However, when multiple asynchronous operations depended
on each other, callbacks became deeply nested, leading to 
Callback Hell (also known as the Pyramid of Doom). 
This made the code difficult to read, maintain, debug, 
and handle errors.
c.To solve these problems, Promises were introduced.
d.A Promise represents the eventual completion (or failure) of 
an asynchronous operation and its resulting value.
e.Promises allow asynchronous code to be written in a 
cleaner way using .then() and .catch(), 
reducing nesting and providing centralized error handling.
f.Although Promises improved readability, 
long chains of .then() calls could still become difficult to follow.
g.Therefore, async/await was introduced as 
syntactic sugar over Promises.
h.It allows asynchronous code to be written in a style that looks similar to synchronous code, making it even more readable and easier to maintain.
Callbacks
   ↓
Callback Hell (deep nesting, difficult maintenance)
   ↓
Promises (.then(), .catch())
   ↓
Async/Await (cleaner syntax over Promises)
